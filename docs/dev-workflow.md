# 开发工作流示例

这个页面可以作为你每个项目的“默认工作说明书”。

## 1. 功能启动前

- 明确需求目标、交付范围和验收方式
- 在文档里先写数据流、页面流或模块边界
- 提前记录风险点，比如依赖、接口时序、兼容性

## 2. 推荐目录约定

```text
src/
├── components/      # 可复用组件
├── features/        # 功能模块
├── services/        # 接口请求层
├── hooks/           # 逻辑复用
├── utils/           # 通用工具
└── pages/           # 页面入口
```

## 3. 开发阶段建议

| 阶段 | 建议动作 | 输出结果 |
| --- | --- | --- |
| 需求拆分 | 先拆页面、状态、接口 | 任务列表 |
| 快速搭建 | 先跑通主链路 | 可访问页面 |
| 细节补全 | 再补异常态和边界 | 更稳定的体验 |
| 自测收尾 | 用清单逐项确认 | 可提交版本 |

## 4. 提交信息样例

```bash
git commit -m "feat: 初始化开发文档站基础页面"
git commit -m "docs: 补充接口说明与组件示例"
git commit -m "fix: 修正文档导航链接"
```

## 5. 发布前检查

- 首页入口是否可达
- 关键文档是否存在死链
- 图片路径是否正确
- 示例代码是否仍然符合当前项目结构
- 待办项是否已显式标注
